# README: 4_quality_control/

---

## Purpose

This folder covers Phase 3 of the project: making sure the Q&As are actually good before they go back to the customer. Annotators peer-review each other's work, and anything that doesn't pass gets kicked back for revision. Nothing leaves the project without clearing this step.

---

## Files

**4_quality_assurance.md**
The internal project document. Describes the full QA pipeline — blind peer review structure, reviewer assignments by sector pair, the 0/1/2 rating scale, disposition logic for each rating outcome, reviewer criteria, the performance bonus ($25 for all 10 Q&As rated 2 on first submission), and the QA budget. Also includes the full project budget summary across all phases, with a suggested total of ~$1,773. Intended for the project lead / Surge team.
