# README: 3_quality_control/

---

## Purpose

This folder covers Phase 3 of the project: making sure the Q&As are actually good before they go back to the customer. Any qualified worker may review any completed row using a 0/1/2 rating scale, and anything that doesn't pass gets kicked back for revision. Nothing leaves the project without clearing this step.

---

## Files

**3_quality_assurance.md**
The internal project document. Describes the full QA pipeline — open per-row peer review, the 0/1/2 rating scale, disposition logic for each rating outcome (including the Cat C rebuttal process and arbitration), reviewer criteria, and the QA budget. Intended for the project lead / Surge team.
